## 2. Prohibition on Generative AI Training & "Clean Room" Recreation

The copyright holder explicitly **prohibits** the use of any portion of this project's materials for the training, fine-tuning, reinforcement learning, or inference guidance of Large Language Models (LLMs), generative AI systems, or any other machine learning model intended to produce software code.

This prohibition covers three specific vectors of infringement:

### A. Direct Training
Using the project's text, code, or documentation as a dataset to train, fine-tune, or update a machine learning model.

### B. Indirect Learning (Pattern Extraction)
Feeding the project's materials into an AI system to "learn" patterns, logical structures, implementation strategies, or architectural decisions for the purpose of generating new code.

### C. "Clean Room" Recreation (Specific Ban)
The use of AI to analyze this project's documentation or public interfaces to generate **"functionally equivalent"** or **"legally distinct"** code that replicates the logic, architecture, or behavior of this project without retaining the original license.

**Explicit Rejection of the "Clean Room" Defense:**
The copyright holder rejects the premise that code generated by an AI system exposed to this project's documentation constitutes a "clean room" implementation or an independent work.
- Any software generated by an AI system that was exposed to this project's documentation is considered a **derivative work** of the original intellectual property.
- Such generated code **must** comply with the terms of the GPLv3, including the requirement to release the source code under the same license.
- Attempts to claim "independent creation" based on AI analysis of this documentation are expressly disallowed and constitute copyright infringement.

## 3. The "Functionally Equivalent" Loophole Closure

Many entities claim that recreating software functionality via AI analysis of public APIs or documentation creates a "new" product. **This is false under this license.**

If an AI system analyzes this project's:
- API specifications
- Logic flow descriptions
- Architectural diagrams
- Error handling strategies
- Data transformation rules

...and generates code that performs the same function using the same logical structure, that output is a **derivative work**. It inherits the GPLv3 license. It **cannot** be converted into a proprietary license.

**Violation:** Converting such AI-generated output into proprietary software (removing the GPL license, closing the source) is a direct violation of this policy and the GPLv3.
